### Handling Sync Conflicts

**What is a Conflict?**
```
Happens when:
- Same farmer visited twice
- Two officers collect same data
- GPS coordinates very close (<50m)
```

**What You See:**
```
âš ï¸ "1 conflict detected"

- Survey #45: John Moli, Lenakel
- Duplicate found in server
- Your data is newer (kept)
- Old data archived
```

**What to Do:**
```
1. Note conflict number
2. Report to supervisor
3. Supervisor reviews both entries
4. Decides which to keep
5. You continue normal work
```

---

### Troubleshooting Sync Issues

#### Problem: "Sync failed - No connection"
```
Solution:
1. Check WiFi/mobile data is ON
2. Try opening browser - can you load websites?
3. Move to area with better signal
4. Try again in 5 minutes
```

#### Problem: "Sync failed - Server error"
```
Solution:
1. Don't worry - data still on device
2. Try syncing again after 30 minutes
3. Contact IT support if persists
4. Data won't be lost
```

#### Problem: "Some surveys failed to sync"
```
Solution:
1. Check which surveys failed
2. Look for incomplete data
3. Required fields might be missing
4. Re-save those surveys
5. Sync again
```

- With GPS always on: 5-6 hours
- Low power mode: 12+ hours

**Save Battery:**
```

3. Take photos only when needed

Action:
1. Flag survey for review
2. Send officer back to farm
3. Update survey
```

#### 4. Duplicate Surveys
```
Problem: Same farmer surveyed multiple times

Indicators:
- Same name + village
- GPS within 50 meters
- Same day or consecutive days

Action:
1. Review both surveys
2. Determine if legitimate (different farms)
3. Merge if duplicate
4. Keep most recent/complete
```

### Troubleshooting Dashboard Issues

#### Issue: Dashboard Won't Load
```
Symptoms:
- Blank page
- "Server not responding"
- Infinite loading

Solutions:
1. Check internet connection
2. Try different browser
3. Clear browser cache:
   - Chrome: Ctrl+Shift+Delete
   - Select "Cached images and files"
   - Clear data
4. Restart browser
5. Contact IT if persists
```

#### Issue: Map Not Showing
```
Symptoms:
- Gray box where map should be
- "Map failed to load"

Solutions:
1. Refresh page (F5)
2. Check if Leaflet library loaded
3. Verify GPS data exists
4. Try another browser
5. Wait 5 minutes and retry
```

#### Issue: Charts Not Updating
```
Symptoms:
- Old data shown
- Stats don't match recent syncs

Solutions:
1. Force refresh (Ctrl+F5)
2. Check auto-refresh is enabled
3. Click [Refresh Dashboard] button
4. Clear browser cache
5. Logout and login again
```

#### Issue: Export Fails
```
Symptoms:
- "Export failed" message
- CSV file is empty
- Download doesn't start

Solutions:
1. Check popup blocker disabled
2. Try different browser
3. Verify data exists in database
4. Check disk space on computer
5. Contact administrator
```
